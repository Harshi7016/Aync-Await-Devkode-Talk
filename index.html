    Why should we go for Async/Await


  • To get away from anti-pattern in code that would happen due to poor structuring of asynchronous callback based javascript code famously known as callback hell
  • To eliminate series of error handling and relying on try/catch method just like other synchronous code
  • Debugging is much simpler. Setting a breakpoint inside a .then block will not move to the next .then because it only steps through synchronous code. But, you can step through await calls as if they were synchronous calls.
